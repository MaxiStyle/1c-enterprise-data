# Руководство по вкладу в проект

Спасибо за интерес к развитию библиотеки 1C EnterpriseData! Это руководство поможет вам внести качественный вклад в проект.

## Как внести вклад

1. **Форкните** репозиторий на GitHub
2. **Клонируйте** свой форк локально:
   ```bash
   git clone https://github.com/your-username/1c-enterprise-data.git
   cd 1c-enterprise-data
   ```
3. **Создайте ветку** для новой функции или исправления:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Требования к коду

### Стандарты кодирования
- Строгое соблюдение **PSR-12**
- Все изменения должны проходить проверку:
  ```bash
  # Проверка стандартов
  ./vendor/bin/phpcs --standard=PSR12 src/
  
  # Автоматическое исправление
  ./vendor/bin/phpcbf --standard=PSR12 src/
  ```

### Анализ кода
- Проект должен проходить проверку **PHPStan** на максимальном уровне:
  ```bash
  ./vendor/bin/phpstan analyse src/ --level=max
  ```

### Тестирование
- При добавлении новой функциональности добавляйте соответствующие тесты
- Убедитесь, что все существующие тесты проходят

## Структура изменений

### Добавление новых типов документов

1. Создайте новый класс сущности в `src/Entities/`
2. Реализуйте соответствующий Builder в `src/Builders/`
3. Добавьте поддержку в `DocumentGenerator.php`
4. Создайте пример использования в `examples/`

### Добавление новых сущностей

1. Создайте класс в соответствующей папке `src/Entities/`
2. Реализуйте необходимые методы
3. Обновите документацию

## Процесс Pull Request

1. **Убедитесь, что код соответствует всем требованиям**
2. **Обновите CHANGELOG.md** с описанием ваших изменений
3. **Создайте Pull Request** с подробным описанием:
   - Что изменено и почему
   - Как тестировались изменения
   - Скриншоты/примеры (если применимо)

### Шаблон описания PR

```markdown
## Описание изменений

Краткое описание того, что делает этот PR.

## Тип изменений

- [ ] Исправление бага
- [ ] Новая функциональность
- [ ] Критическое изменение
- [ ] Документация
- [ ] Рефакторинг

## Как тестировалось

Описание тестов, которые вы провели.

## Чек-лист

- [ ] Код соответствует PSR-12
- [ ] Проходит анализ PHPStan
- [ ] Добавлены/обновлены тесты
- [ ] Обновлена документация
- [ ] CHANGELOG.md обновлён
```

## Сообщения коммитов

Используйте [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` - новая функциональность
- `fix:` - исправление бага
- `docs:` - изменения в документации
- `style:` - форматирование кода
- `refactor:` - рефакторинг
- `test:` - добавление тестов
- `chore:` - изменения в процессе сборки

Примеры:
```
feat: добавить поддержку документа "Акт выполненных работ"
fix: исправить генерацию XML для пустых полей
docs: обновить примеры использования
```

## Поддержка

Если у вас есть вопросы:
- Создайте Issue с меткой `question`
- Опишите проблему максимально подробно
- Приложите пример кода, если речь о баге

## Лицензия

Внося вклад, вы соглашаетесь с тем, что ваши изменения будут опубликованы под лицензией MIT проекта.
